### 修复：昵称区域点击报错 + 显示为输入框问题

用户反馈点击头像旁边的昵称区域会报错（`inputId`/`hoverClass` 等非法属性），且名称会显示为输入框。

**根因**：`<input type="nickname">` 默认可交互，点击即进入编辑态，看起来不像纯文本。

**修复方案**：
- 默认用 `<view>` 显示昵称纯文本，点击后切换到 `<input type="nickname">` 编辑
- 新增 `editingName` 状态变量控制 view/input 切换
- 新增 `startEditName()` 方法，点击 view 时设为 true 显示 input
- `onNicknameInput` 完成后设回 false，恢复纯文本显示
- `applyProfile` 登出时重置 editingName 为 false
- profile.ts 和 profile.js 同步更新

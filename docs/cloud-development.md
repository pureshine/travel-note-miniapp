# 临时出逃云开发接入说明

当前云环境 ID：

```txt
cloud1-d2gse79u56ad69a8a
```

## 1. 创建数据库集合

在微信开发者工具中进入「云开发」：

1. 打开「数据库」
2. 新建集合：`shared_trips`
3. 新建集合：`trip_invitations`
4. 权限建议先选「仅创建者可读写」

`shared_trips` 用来保存可多人共享的旅行数据。每个旅行文档里会记录成员 openid，好友加入后会读取同一份旅行。

`trip_invitations` 用来保存好友邀请。当前邀请有效期为 7 天。

## 2. 配置云存储

日程图片会上传到云存储，路径格式类似：

```txt
trip-images/旅行ID/时间戳-图片ID.jpg
```

建议在「云开发」->「存储」里确认权限：

1. 上传：登录用户可上传
2. 读取：建议设置为「所有用户可读」
3. 删除：先保留管理员或创建者可删

如果读取权限过严，好友加入共享旅行后可能能看到日程文字，但看不到图片。

## 3. 上传云函数

在开发者工具左侧找到 `cloudfunctions`：

1. 右键 `login`
2. 选择「上传并部署：云端安装依赖」
3. 右键 `syncTrips`
4. 选择「上传并部署：云端安装依赖」

`login` 用来获取当前微信用户的 `openid`。
`syncTrips` 用来同步共享旅行、生成邀请和接受好友邀请。

## 4. 小程序端测试

1. 编译小程序
2. 进入「我的」
3. 点击「微信登录」
4. 登录成功后会自动同步当前旅行数据
5. 点击「生成邀请」后，再点击「分享给好友」
6. 好友打开分享卡片后，登录微信即可加入同一份旅行

## 5. 如果数据库没有数据

按下面顺序排查：

1. 先确认已经创建集合：`shared_trips`、`trip_invitations`
2. 确认 `login`、`syncTrips` 都已经「上传并部署：云端安装依赖」
3. 在小程序进入「我的」，点击「微信一键登录」
4. 登录后新增或修改一条日程、备忘或消费，等待 2 秒
5. 打开云开发数据库，查看 `shared_trips`
6. 如果仍然没有数据，看「我的」页共享旅行卡片里的同步提示
7. 也可以打开开发者工具 Console，搜索 `syncTrips`

注意：没有微信登录时，数据只保存在本机缓存，不会上传数据库。

## 6. 当前同步策略

当前版本采用轻量共享同步：

- 自动同步：登录后，本机新增或修改旅行、日程、备忘、消费时会延迟约 1.6 秒自动上传云端
- 好友共享：按旅行计划共享，加入后两个人维护同一份日程、备忘和消费
- 图片同步：日程图片上传到云存储，日程数据里保存云文件 ID

当前冲突策略是后同步者覆盖同一份旅行。后续可以继续升级为按日程、备忘、消费单条记录合并。
